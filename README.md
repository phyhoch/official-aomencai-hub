# official-aomencai-hub

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容不针对任何单一域名或具体网站，仅作为静态资源的集中存储与展示用途。

仓库以“归档”为核心目的，保存不同时期生成的 HTML 文件，便于后续查阅、引用或版本回溯。

## 目录结构

```
official-aomencai-hub/
├── index.html          # 仓库首页 / 导航页
├── pages/              # 存放各独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md
```

- **`pages/`**：所有独立 HTML 页面均放置于此目录下，文件名应具有辨识度。
- **`assets/`**：存放页面引用的公共静态资源，避免重复上传。
- **`index.html`**：可选，用于提供页面列表或快速导航。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，不依赖外部动态服务。
- 归档的页面可能包含样式、脚本或图片引用，所有依赖资源应尽量放置于 `assets/` 目录中，以保证页面可离线查看。
- 页面名称应保持清晰、一致，建议使用语义化的英文或拼音命名，避免歧义。
- 历史版本可通过 Git 提交记录回溯，不建议在同一路径下保留多个版本的同名文件。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有必要，同步更新 `assets/` 中的资源。
- 删除页面：仅移除对应文件，不推荐覆盖或修改历史提交。
- 修改页面：直接编辑文件并提交，建议在 commit message 中简要说明修改原因。
- 公共资源：若多个页面共用同一资源（如 CSS 框架），请优先放入 `assets/` 以避免冗余。

## 许可

本仓库内容仅供归档与展示，不构成任何商业或服务承诺。如需引用或分发，请自行评估内容的适用性。
